    # pwd指令

    1. 基本语法：【pwd】

    2. 功能描述：显示当前工作目录的绝对路径

    3. 应用实例：显示当前工作目录的绝对路径

      [root@hadoop1 ~]# pwd
/root

    # ls指令

    1. 基本语法：【ls[选项][目录或是文件]】

    2. 常用选项：-a 显示当前目录所有的文件和目录，包括隐藏的

      ​ -l 以列表的方式显示信息

    3. 应用实例：查看当前目录的所有内容信息

      [root@hadoop1 ~]# ls
...
[root@hadoop1 ~]# ls -l
...
[root@hadoop1 ~]# ls -al
...

    # cd指令

    1. 基本语法：【cd[参数]】

    2. 功能描述：切换到指定目录

    3. 常用参数：[cd~或cd : 回到自己的家目录]，[cd .. 回到当前目录的上一级目录]

    4. 应用实例：

      使用绝对路径切换到/root目录

      [root@hadoop1 home]# cd /root/
[root@hadoop1 ~]#

      使用相对路径到/root目录，假设现在是/usr/lib

      [root@hadoop1 lib]# ../../root/
[root@hadoop1 ~]#

      表示回到当前目录的上一级目录

      [root@hadoop1 lib]# cd ..

      回到家目录

      [root@hadoop1 home]# cd
[root@hadoop1 ~]#

    # mkdir指令

    1. 基本语法：【mkdir [选项] 要创建的目录】

    2. 功能描述：创建目录make directory

    3. 常用选项：-p 创建多级目录

    4. 应用实例：

      创建一个目录/home/dog

      [root@hadoop1 ~]# mkdir /home/dog
[root@hadoop1 ~]#
[root@hadoop1 ~]# cd /home/
[root@hadoop1 home]# ls
dog hello tiger cat

      创建多级目录/home/animal/tiger

      [root@hadoop1 home]# mkdir -p /home/animal/tiger
[root@hadoop1 home]# ls
animal dog hello tiger cat
[root@hadoop1 home]# cd animal/
[root@hadoop1 animal]# ls
tiger

    # rmdir指令

    1. 基本语法：【rmdir [选项] 要删除的空目录】

    2. 功能描述：删除空目录

    3. 应用实例：删除一个目录/home/dog

      [root@hadoop1 home]# lsanimal dog hello tiger cat[root@hadoop1 home]# rmdir /home/dog[root@hadoop1 home]# lsanimal hello tiger cat

    4. 使用细节：删除的是空目录，如果还目录下有内容是无法删除的，如果需要删除非空目录，需要使用rm-rf要删除的目录

      [root@hadoop1 dog]# rmdir /home/dogrmdir: 删除 "/home/dog" 失败：目录非空[root@hadoop1 dog]# rm -rf /home/dog

    # touch指令

    1. 基本语法：【touch 文件名称】

    2. 功能描述：创建空文件

    3. 应用实例：创建一个空文件hello.txt

      [root@hadoop1 home]# touch hello.txt[root@hadoop1 home]# lsanimal hello hello.txt tiger[root@hadoop1 home]# touch ok1.txt ok2.txt[root@hadoop1 home]# lsanimal hello hello.txt ok1.txt ok2.txt tiger

    # cp指令

    1. 基本语法：【cp [选项] source dest】

    2. 功能描述：拷贝文件到指定目录

    3. 常用选项：-r 递归复制整个文件夹

    4. 应用实例：

      将/home/aaa.txt拷贝到/home/bbb目录下[拷贝单个文件]

      [root@hadoop1 home]# touch aaa.txt[root@hadoop1 home]# lsaaa.txt[root@hadoop1 home]# mkdir bbb[root@hadoop1 home]# lsaaa.txt bbb[root@hadoop1 home]# cp aaa.txt bbb/[root@hadoop1 home]# cd bbb/[root@hadoop1 bbb]# lsaaa.txt

      递归复制整个文件夹，将/home/test整个目录拷贝到/home/zwj目录

      [root@hadoop1 home]# cd test/
[root@hadoop1 test]# ls
a.txt b.txt c.txt
[root@hadoop1 test]# cd ..
[root@hadoop1 home]#
[root@hadoop1 home]# cp -r test/ zwj/
[root@hadoop1 home]# cd zwj/
[root@hadoop1 zwj]# ls
test
[root@hadoop1 zwj]# cd test/
[root@hadoop1 test]# ls
a.txt b.txt c.txt

    5. 使用细节：强制覆盖不提示\cp

      //当发现目录下有相同文件，会提示你是否覆盖
[root@hadoop1 home]# cp -r test/ zwj/
cp: 是否覆盖"zwj/test/c.txt"? y
cp: 是否覆盖"zwj/test/b.txt"? y
cp: 是否覆盖"zwj/test/a.txt"? y

//强制覆盖原来的文件，不会提示
[root@hadoop1 home]# /cp -r test/ zwj/

    # rm指令

    1. 基本语法：【rm [选项] 要删除的文件或目录】

    2. 功能描述：移除[删除]文件或目录

    3. 常用选项：-r递归删除整个文件夹，-f强制删除不提示

    4. 应用实例：

      将/home/aaa.txt删除

      [root@hadoop1 home]# rm aaa.txt
rm: 是否删除普通空文件 "aaa.txt"? y

      递归删除整个文件夹/home/bbb

      [root@hadoop1 home]# rm -rf bbb/

      强制删除不提示

      [root@hadoop1 home]# rm -f ok1.txt

    # mv指令

    1. 基本语法：【mv /temp/movefile/targetFolder】【mv oldNameFile newNameFile】

    2. 功能描述：移动文件与目录或重命名

    3. 应用实例：

      将/home/aaa.txt文件重命名为pig.txt

      [root@hadoop1 home]# lsaaa.txt[root@hadoop1 home]# mv aaa.txt pig.txt[root@hadoop1 home]#lspig.txt

      将/home/pig.txt文件移动到/root目录下

      [root@hadoop1 home]# mv pig.txt /root/[root@hadoop1 home]# cd /root/[root@hadoop1 ~]# lspig.txt

    # cat指令

    1. 基本语法：【cat [选项] 要查看的文件】

    2. 功能描述：查看文件内容，是以只读的方式打开

    3. 常用选项：-n显示行号

    4. 应用实例：/etc/profile文件内容，并显示行号

      [root@hadoop1 ~]# cat -n /etc/profile | more

    5. 使用细节：cat只能浏览文件，而不能修改文件，为了浏览方便，一般会带上管道命令|more

    # more指令

    1. 基本语法：【more 要查看的文件】

    2. 功能描述：more指令是一个基于VI编辑器的文本过滤器，它以全屏幕的方式按页显示文本文件的内容

    3. 应用实例：

      采用more查看文件/etc/profile

      [root@hadoop1 ~]# more /etc/profile

    快捷键：

    操作 功能说明
    空白键（space） 向下翻一页
    Enter 向下翻一行
    q 立刻离开more，不再显示该文件内容
    Ctrl+F 向下滚动一屏
    Ctrl+B 返回上一屏
    = 输出当前行的行号
    :f 输出文件名和当前行的行号

    # less指令

    1. 基本语法：【less 要查看的文件】

    2. 功能描述：分屏查看文件内容，比more更强大，支持各种显示终端，less指令在现实文件内容时，并不是一次将整个文件加载之后才显示，而是跟据显示需要加载内容，对于显示大型文件具有较高的效率

    3. 应用实例：采用less查看一个大文件/opt/金庸-射雕英雄传txt精校版.txt

    [root@hadoop1 ~]# cd/opt/[root@hadoop1 opt]# less 金庸-射雕英雄传txt精校版.txt

    快捷键：

    操作 功能说明
    空白键 向下翻动一页
    [pagedown] 向下翻动一页
    [pageup] 向上翻动一页
    /字串 向下搜寻[字串] n向下查找 N向上查找
    ?字串 向上搜寻[字串] n向下查找 N向上查找
    q 离开less这个程序

    # >和>>指令

    1. 【>】输出重定向，会将原来的文件内容覆盖

    2. 【>>】不会覆盖原来文件的内容，而是追加到文件的尾部

    3. 基本语法：

      ls -l>文件 列表内容写入文件a.txt(覆盖)，如果该文件不存在就创建该文件

      ls -al>>文件 列表的内容追加到文件aa.txt的末尾

      cat 文件1 > 文件2 将文件1的内容覆盖到文件2

      echo "内容" >> 文件

    4. 应用案例：

      将/home目录下的文件列表写入到/home/info.txt中

      [root@hadoop1 home]# ls -l /home/ > /home/info.txt

      将当前日历信息追加到/home/mycal文件中

      [root@hadoop1 home]# cal >> /home/mycal

    # echo指令

    1. 基本语法：【echo [选项] [输出内容]】

    2. 功能描述：输出内容到控制台

    3. 应用实例：

      输出环境变量，输出当前的环境路径，输出hello,world!

      [root@hadoop1 home]# echo $PATH

    # head指令

    1. 基本语法1：【head 文件】查看文件头5行内容

    2. 基本语法2：【head -n 5文件】查看文件头5行内容，5可以是任意行数

    3. 功能描述：用于显示文件的开头部分内容，默认情况下head指令显示文件的前10行内容

    4. 应用实例：查看/etc/profile的前面5行代码

      [root@hadoop1 home]# head -n 5 /etc/profile

    # tail指令

    1. 基本语法1：【tail 文件】查看文件后10行内容

    2. 基本语法2：【tail -n 5 文件】查看文件后5行内容，5可以是任意行数

    3. 基本语法3：【tail -f 文件】实时追踪该文档的所有更新，工作经常使用

    4. 功能描述：用于输出文件中尾部的内容，默认情况下tail指令显示文件的后10行内容

    5. 应用实例：

      查看/etc/profile最后5行的代码

      [root@hadoop1 home]# tail -n 5 /etc/profile

      实时监控mydate.txt，追加日期

      [root@hadoop1 home]# tail -f mydata.txt

    # ln指令

    1. 基本语法：【ln -s [原文件或目录][软连接名]】

    2. 功能描述：给原文件创建一个软连接，软连接也叫符号连接，类似于windows里的快捷方式，主要存放了连接其他文件的路径

    3. 应用实例：

      在/home目录下创建一个软连接linkToRoot，连接到/root目录

      [root@hadoop1 home]# ln -s /root linkToRoot

      删除软连接linkToRoot

      [root@hadoop1 home]# rm -rf linkToRoot

    4. 细节说明：当我们使用pwd指令查看目录时，仍然看到的是软连接所在目录

    # history指令

    1. 基本语法：【history】

    2. 常用选项：查看已执行过历史命令

    3. 应用实例：

      显示所有的历史命令

      [root@hadoop1 home]# history

      显示最近使用过的10个指令

      [root@hadoop1 home]# history 10

      执行历史编号为5的指令

      [root@hadoop1 home]# !5
