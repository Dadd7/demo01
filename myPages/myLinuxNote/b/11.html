    进程管理

    大段 1/23/2022 linux

    # 基本介绍

    1. 在Linux中，每个执行的程序(代码)都成为一个进程，每一个进程都分配一个ID号
    2. 每一个进程，都会对应一个父进程，而这个父进程可以复制多个子进程，例如www服务器
    3. 每个进程都可能以两种方式存在：前台和后台。所谓的前台进程就是用户目前的屏幕上可以操作的，后台进程则是实际在操作。屏幕上无法看到的进程，通常实用后台方式执行
    4. 一般系统的服务都是以后台进程的方式存在的，而且都会常驻在系统中，直到关机才结束

    # [ps] 显示系统执行的进程

    1. ps命令是用来查看目前系统中，有哪些正在执行，以及他们执行的情况

    2. 【ps -a】显示当前终端的所有进程信息

      【ps -u】以用户的格式显示进程信息

      【ps -x】显示后台进程运行的参数

    3. USER PID %CPU %MEM VSZ RSS TTY STAT START TIME COMMAND
      用户名 进程id 占用的cpu 占用内存 使用的虚拟内存 使用物理内存情况 使用的终端 进程状态 启动时间 占用cpu总时间 进程执行的命令行

    5. STAT 进程状态，其中，S 睡眠，表示该进程是会话的先导进程；N 表示进程拥有比普通优先级更低的优先级；R 正在运行；D 短期等待；Z 僵死进程；T 被跟踪或者被停止

    # 终止进程

    若是某个进程执行一半需要停止时，或时已消了很大的系统资源时，此时可以考虑停止该进程。使用kill命令来完成此任务

    【kill [选项] 进程号】通过进程号杀死进程 -9表示强迫进程立即停止

    【killall 进程名称】通过进程名称杀死进程，也支持通配符，这在系统因负载过大而变得很慢时才有用

    // 踢掉某个非法登录用户
[root@hadoop1 桌面]# ps -aux | grep sshd
Warning: bad syntax, perhaps a bogus '-'? See /usr/share/doc/procps-3.2.8/FAQ
jack	4010  0.6  0.0  102084  1844 ?		S   18:49  0:00  sshd:  jack@pts
[root@hadoop1 桌面]# kill 4010

//终止远程登录服务sshd,在适当时候再次重启sshd服务
[root@hadoop1 桌面]# ps -aux | grep sshd
Warning: bad syntax, perhaps a bogus '-'? See /usr/share/doc/procps-3.2.8/FAQ
root     3098  0.0  0.0  66236  1200 ?        Ss  18:08  0:00  /user/sbin/sshd
[root@hadoop1 桌面]# kill 3908

//终止多个gedit编辑器
[root@hadoop1 桌面]# killall gedit

    # 查看进程树

    【pstree [选项]】可以更加直观的来看进程信息

    选项：-p 显示进程的PID；-u 显示进程所属用户

    //树状的形式显示进程的pid
[root@hadoop1 桌面]# pstree -p
//树状形式显示进程的用户id
[root@hadoop1 桌面]# pstree -u
