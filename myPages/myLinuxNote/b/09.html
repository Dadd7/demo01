    磁盘分区和挂载

    大段 1/23/2022 linux

    # 分区基础知识

    mbr分区 gtp分区
    最多支持四个主分区 支持无限多个主分区（但操作系统可能限制，windows最多128个分区）
    系统只能安装在主分区 最大支持18EB的大容量（1EB=1024PB，1PB=1024TB）
    扩展分区要占一个主分区 Windows7 64位以后支持gtp
    MBR最大只支持2TB，但拥有更好的兼容性

    # Linux分区

    1. 原理介绍

      Linux来说，无论有几个分区，分给哪一目录使用，它归根到底就只有一个根目录，一个独立且唯一的文件，Linux中每个文件都是用来组成整个文件系统的一部分

      Linux采用了一种叫“载入”的处理方式，它的整个文件系统中包含了一整套的文件和目录，且将一个分区和一个目录联系起来，这时要载入的一个分区将使它的存储空间在一个目录下获得

    2. 硬盘说明

      Linux硬盘分IDE硬盘和SCSI硬盘，目前基本上是SCSI硬盘

      对于IDE硬盘，驱动器标识符为“hdx~”，其中“hd”表明分区所在设备的类型，这里是指IDE硬盘了，“x”为盘号（a为基本盘，b为基本从属盘，c为辅助主盘，d为辅助从属盘），“~”代表分区，前四个分区用数组1到4表示，它们是主分区或扩展分区，从5开始就是逻辑分区。例，hda3表示为第一个IDE硬盘上的第三个主分区或扩展分区

      对于SCSI硬盘则标识为“sdx~”，SCSI硬盘是用“sd”来标识分区所在设备的类型的，其余则和IDE硬盘的表示方法一样

    3. 使用lsblk指令查看当前系统的分区情况

      FSTYPE 分区情况；LABEL 分区类型；UUID 唯一标识分区的40位不重复的字符串；MOUNTPOINT挂载点

      [root@hadoop1 ~]# lsblk -f
NAME	FSTYPE LABEL UUID							 MOUNTPOINT
sr0
sda
├─sda1 ext4		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /boot
├─sda2 swap		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx [swap]
└─sda3 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /
[root@hadoop1 ~]# lsblk
NAME	MAJ:MIN	 RM	   SIZE RO	TYPE  MOUNTPOINT
sr0		 11:0 	  1	  1024M  0  rom
sda		  8:0	  0     20G  0  disk
├─sda1	  8:1  	  0    200M  0  part  /boot
├─sda2	  8:2     0      2G  0  part  [SWAP]
└─sda3    8:3     0   17.8G  0  part  /

    # 增加一块硬盘

    1. 点击WMware左上角【虚拟机】，点击【设置】

    2. 来到【虚拟机设置】窗口，点击下方的【添加】

    3. 选择【硬盘】，点击【下一步】

    4. 虚拟磁盘类型选择【SCSI(S)】，点击【下一步】

    5. 选择【创建新虚拟磁盘】，点击【下一步】

    6. 最大磁盘大小【2G】(随意)，选择【将虚拟磁盘拆分成多个文件(M)】，点击【下一步】

    7. 文件名【atguiguCentOS-0.vmdk】，这个名字应该是啥就让是啥就行，点击【完成】

    8. 又回到虚拟机设置窗口，点击【确定】

    9. 终端输入【lsblk】看不到新磁盘sdb1，是因为需要重启才行

    10. 重启后，在终端【lsblk -f】，发现有这个硬盘了，就是没有分区和格式化

      [root@hadoop1 ~]# lsblk -f
NAME	FSTYPE LABEL UUID							 MOUNTPOINT
sr0
sda
├─sda1 ext4		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /boot
├─sda2 swap		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx [swap]
└─sda3 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /
sdb1

    # 分区

    1. 终端输入【fdisk /dev/sdb】回车，出现【Command <m for help>:】你需要什么帮助

    2. 在帮助后面输入【m】，看到了【n add a new partition】【w write table to disk and exit】，输入【n】

    3. 出现【p primary partition(1-4)】，输入【p】

    4. Partition number (1-4): 1
First cylinder (1-261, default 1):
Using default value 1
Last cylinder, +cylinders or +size{k,M,G} (1-261, default 261):
Using default value 261

Command <m for help>: w

    5. 再看分区

      [root@hadoop1 ~]# lsblk -f

    # 格式化

    1. 输入【mkfs -t ext4 /dev/sdb1】，回车，就成功了

    2. 再看分区，这个磁盘就可以用了

      [root@hadoop1 ~]# lsblk -f
NAME	FSTYPE LABEL UUID							 MOUNTPOINT
sr0

├─sda1 ext4		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /boot
├─sda2 swap		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx [swap]
└─sda3 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /
sdb
└─sdb1 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

    # 挂载

    1. 需要挂在到newdisk，先创建newdisk【mkdir /home/newdisk】

    2. 挂载【[root@hadoop1 home]# mount /dev/sdb1 /home/newdisk】

    3. [root@hadoop1 ~]# lsblk -f
NAME	FSTYPE LABEL UUID							 MOUNTPOINT
sr0
sda
├─sda1 ext4		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /boot
├─sda2 swap		xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx [swap]
└─sda3 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /
sdb
└─sdb1 ext4 	xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx /home/newdisk

    4. 这时临时挂载，重启后，这种挂载联系就没了，如果想一直保持联系，需要设置自动挂载

    # 自动挂载

    1. 打开【vim /etc/fstab】

    2. 写入挂【/dev/sdb1 /home/newdisk ext4 defaults 0 0】，保存

      #
# /etc/fstab
# Created by anaconda on Sat Mar 17 21:07:00 2018
#
# Accessible filesystems, by reference, are maintained under '/dev/disk'
# See man pages fstab(5), findfs(8), mount(8) and/or blkid(8) for more info
#
/dev/sdb1                                /home/newdisk     ext4  defaults  0 0
...

    3. 写【[root@hadoop1 home]# mount -a】让挂载生效

    4. 【reboot】重启

    5. 要是取消挂载，就【umount /dev/sdb1】

    # 磁盘情况查询

    1. 【df -h】查询系统整体磁盘使用情况

    2. 【du-h /目录】查询指定目录的磁盘占用情况

      查询指定目录的磁盘占用情况，默认为当前目录

      -s 指定目录占用大小总汇；-h 带计量单位；-a 含文件；

      --max-depth=1 子目录深度; -c 列出明细的同时，增加汇总值

      //查询/opt目录的磁盘占用情况，深度为1
[root@hadoop1 home]# du -ach --max-depth=1 /opt

    3. 工作实用指令

      【ls -l /home | grep "^-" | wc -l】统计/home文件夹下文件的个数

      【ls -l /home | grep "^d" | wc -l】统计/home文件夹下目录的个数

      【ls -lR /home | grep "^-" | wc -l】统计/home文件夹下文件的个数，包括子文件夹里的

      【ls -lR /home | grep "^d" | wc -l】统计/home文件夹下目录的个数，包括子文件夹里的

      【tree】以数状显示目录结构 (需要先安装yum instal tree)
